## 👥 Employees (직원 데이터)

### 테이블 역할
직원의 기본 정보, 스킬셋, 경력, 학력, 자격증을 관리하는 핵심 테이블입니다. 프로젝트 매칭 시 직원의 역량을 평가하고, 적합한 인력을 추천하는 데 사용됩니다.

### 저장 데이터 구조
- **기본 정보**: user_id, 이름, 이메일, 역할, 경력 연수
- **학력**: 학위, 대학교
- **스킬**: 기술명, 숙련도 레벨 (Beginner/Intermediate/Expert)
- **자격증**: 보유 자격증 목록
- **배정 정보**: 현재 프로젝트 배정 날짜

### 현재 데이터 현황
- **총 직원 수**: 300명
- **평균 경력**: 8.5년 (범위: 2년 ~ 15년)
- **역할 다양성**: 20개 이상의 직무 역할 (Scrum Master, Backend Developer, Data Engineer 등)
- **학위 분포**: Computer Science(40명), Mathematics(31명), Software Engineering(31명) 등

### 활용 목적
- 프로젝트 요구사항에 맞는 인력 추천
- 스킬 갭 분석 및 교육 계획 수립
- 직원 역량 기반 팀 구성 최적화

## 📁 Projects (프로젝트 데이터)

### 테이블 역할
과거 및 진행 중인 프로젝트의 정보를 관리하며, 프로젝트별 팀 구성, 요구 기술, 산업 분야를 추적합니다. 신규 프로젝트 팀 구성 시 유사 프로젝트 경험을 기반으로 최적의 인력을 추천하는 데 활용됩니다.

### 저장 데이터 구조
- **프로젝트 정보**: project_id, 프로젝트명, 고객 산업, 상태
- **기간**: 시작일, 종료일
- **요구사항**: 필요 기술 및 역량 목록
- **팀 구성**: 역할별 배정된 직원 ID 목록 (team_composition)
- **예산 및 규모**: 프로젝트 규모 정보

### 현재 데이터 현황
- **총 프로젝트**: 100개
- **상태 분포**: 완료 50개(50%), 진행 중 50개(50%)
- **산업 분포**: E-commerce(27개), Finance(26개), Manufacturing(24개), Healthcare(23개)
- **평균 팀 크기**: 5.7명/프로젝트
- **주요 역할 수요**: DevOps Engineer(138회), Backend Developer(105회), Full Stack Developer(94회)

### 활용 목적
- 유사 프로젝트 경험 기반 인력 추천
- 산업별/기술별 프로젝트 트렌드 분석
- 직원의 프로젝트 이력 및 경험 추적
- 팀 구성 패턴 학습 및 최적화

## 💬 MessengerLogs (메신저 로그)

### 테이블 역할
직원 간 메신저 커뮤니케이션 패턴을 분석하여 협업 관계와 소통 빈도를 파악합니다. 이 데이터는 직원 간 친밀도(EmployeeAffinity) 계산의 기초 자료로 활용되며, 효과적인 팀 구성을 위한 인사이트를 제공합니다.

### 저장 데이터 구조
- **메시지 정보**: log_id, 발신자 ID, 수신자 ID, 타임스탬프
- **대화 스레드**: conversation_thread_id (대화 그룹 추적)
- **메시지 내용**: 익명화된 메시지 내용 ([ANONYMIZED])
- **응답 패턴**: 응답 시간 (분 단위)

### 현재 데이터 현황
- **총 메시지**: 2,008개
- **기간**: 2024년 12월 ~ 2025년 6월 (약 7개월)
- **참여 직원**: 300명 전원
- **대화 스레드**: 다수의 1:1 및 그룹 대화

### 활용 목적
- 직원 간 협업 빈도 및 소통 패턴 분석
- 팀 내/팀 간 커뮤니케이션 네트워크 시각화
- 친밀도 점수 계산을 위한 기초 데이터
- 비공식 협업 관계 파악 (공식 프로젝트 외)

## 🤝 EmployeeAffinity (직원 친밀도)

### 테이블 역할
두 직원 간의 협업 친밀도를 정량화하여 저장합니다. 프로젝트 협업 이력, 메신저 소통 빈도, 회사 이벤트 참여 등 다양한 요소를 종합하여 친밀도 점수를 계산합니다. 팀 구성 시 협업 시너지가 높은 조합을 추천하는 데 활용됩니다.

### 저장 데이터 구조
- **관계 정보**: affinity_id, employee_1, employee_2
- **종합 점수**: overall_affinity_score (0-100점)
- **프로젝트 협업**: 공동 참여 프로젝트 목록, 협업 기간, 같은 팀 여부
- **개인적 친밀도**: 휴가일/급여일 연락 빈도, personal_score
- **이벤트 참여**: 함께 참여한 회사 이벤트 목록
- **업데이트 날짜**: 마지막 계산 일자

### 현재 데이터 현황
- **총 레코드**: 5개 (샘플 데이터)
- **평균 점수**: 57.6점
- **점수 분포**: 높음(70+) 2개, 중간(40-69) 2개, 낮음(<40) 1개
- **계산 기준**: 프로젝트 협업 이력 + 메신저 소통 + 이벤트 참여

### 활용 목적
- 협업 시너지가 높은 직원 조합 추천
- 팀 구성 시 갈등 가능성 최소화
- 멘토-멘티 매칭 지원
- 조직 내 비공식 네트워크 분석

### 확장 필요성
현재 5개의 샘플 데이터만 존재하므로, 300명 직원 간 조합(약 44,850개)으로 확장이 필요합니다.

## 🎉 CompanyEvents (회사 이벤트)

### 테이블 역할
회사 주최 이벤트 및 참가자 정보를 기록합니다. 공식 업무 외 직원 간 교류 기회를 추적하여 친밀도 계산에 활용되며, 조직 문화 및 팀 빌딩 활동의 효과를 분석하는 데 사용됩니다.

### 저장 데이터 구조
- **이벤트 정보**: event_id, 이벤트명, 날짜, 장소
- **이벤트 타입**: Company-wide, Team Building, Recreation, Education
- **참가자**: 참가 직원 ID 목록
- **설명**: 이벤트 상세 설명

### 현재 데이터 현황
- **총 이벤트**: 6개 (2024년 1월 ~ 12월)
- **타입 분포**: 전사 행사 3개, 레크리에이션 1개, 팀 빌딩 1개, 교육 1개
- **평균 참가자**: 6.3명
- **주요 이벤트**: 신년회(8명), 송년회(10명), 여름 MT(6명) 등

### 이벤트 목록
| 날짜 | 이벤트명 | 타입 | 참가자 |
|------|----------|------|--------|
| 2024-01-20 | 2024 신년회 | Company-wide | 8명 |
| 2024-03-15 | 팀 빌딩 워크샵 | Team Building | 5명 |
| 2024-05-10 | 회사 창립기념일 | Company-wide | 5명 |
| 2024-07-22 | 여름 MT | Recreation | 6명 |
| 2024-09-18 | 기술 세미나 | Education | 4명 |
| 2024-12-20 | 송년회 | Company-wide | 10명 |

### 활용 목적
- 직원 간 비공식 네트워크 형성 추적
- 친밀도 점수 계산 시 이벤트 참여 가중치 반영
- 조직 문화 활동 효과 분석
- 팀 빌딩 프로그램 기획 지원

## ⚠️ 빈 테이블

### EmployeeEvaluations (직원 평가)

**테이블 역할**  
직원의 성과 평가, 역량 평가, 프로젝트 기여도를 기록하고 관리합니다. 평가 결과는 인력 추천 시 우선순위 결정과 직원 성장 계획 수립에 활용됩니다.

**저장 예정 데이터 구조**
- **평가 정보**: evaluation_id, 직원 ID, 평가자 ID, 평가 날짜
- **평가 항목**: 기술 역량, 협업 능력, 리더십, 문제 해결 능력
- **점수**: 항목별 점수 및 종합 점수
- **피드백**: 정성적 평가 및 개선 제안
- **상태**: 제출 상태, 승인 여부

**활용 목적**
- 고성과자 우선 배정
- 직원 성장 경로 추적
- 교육 필요 영역 식별
- 승진 및 보상 의사결정 지원

**현재 상태**: 데이터 없음 (입력 필요)

### TechTrends (기술 트렌드)

**테이블 역할**  
시장의 기술 트렌드와 수요를 추적하여, 직원 교육 방향 설정과 프로젝트 수주 전략 수립에 활용됩니다. 트렌드 점수를 기반으로 미래 수요가 높은 기술을 보유한 인력을 우선 추천합니다.

**저장 예정 데이터 구조**
- **기술 정보**: tech_name, 카테고리 (언어/프레임워크/플랫폼)
- **트렌드 점수**: 시장 수요 지수 (0-100)
- **성장률**: 전년 대비 성장률
- **관련 산업**: 주요 활용 산업 분야
- **업데이트 날짜**: 트렌드 데이터 갱신 일자

**활용 목적**
- 미래 수요 기술 보유 인력 우선 추천
- 직원 교육 프로그램 우선순위 결정
- 신규 채용 시 필요 기술 식별
- 프로젝트 수주 전략 수립

**현재 상태**: 데이터 없음 (입력 필요)
