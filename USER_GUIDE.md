## 시스템 개요

HR Resource Optimization System은 AI 기반 인력 배치 최적화 플랫폼입니다. 이 시스템은 다음과 같은 주요 기능을 제공합니다:

- **직원 관리**: 직원 정보 등록 및 관리
- **프로젝트 관리**: 프로젝트 등록 및 인력 배정
- **AI 인력 추천**: 프로젝트에 최적의 인력 추천
- **평가 관리**: 직원 이력서 분석 및 평가 워크플로우
- **도메인 분석**: 신규 사업 도메인 확장 기회 분석
- **대시보드**: 실시간 인력 현황 및 메트릭 조회

## 시작하기

### 시스템 접속

1. 웹 브라우저에서 시스템 URL에 접속합니다
2. AWS IAM 자격 증명으로 로그인합니다
3. 메인 대시보드가 표시됩니다

### 메인 화면 구성

시스템은 다음과 같은 주요 메뉴로 구성되어 있습니다:

- **대시보드**: 전체 현황 및 주요 메트릭
- **인력 관리**: 직원 목록 및 등록
- **프로젝트 관리**: 프로젝트 목록 및 등록
- **인력 추천**: AI 기반 프로젝트 투입 인력 추천
- **평가 관리**: 직원 평가 및 이력서 분석
- **도메인 분석**: 신규 도메인 확장 분석

## 직원 관리

### 직원 목록 조회

1. 좌측 메뉴에서 **"인력 관리"**를 클릭합니다
2. 등록된 모든 직원 목록이 표시됩니다
3. 각 직원 카드에는 다음 정보가 표시됩니다:
   - 이름 및 직급
   - 보유 기술 스택
   - 경력 연수
   - 현재 프로젝트 배정 상태


### 새로운 직원 등록

#### 단계별 가이드

**1단계: 등록 모달 열기**
- 인력 관리 페이지 우측 상단의 **"+ 신규 직원 등록"** 버튼을 클릭합니다
- 직원 등록 모달 창이 나타납니다

**2단계: 기본 정보 입력**

필수 입력 항목:
- **이름**: 직원의 전체 이름
- **이메일**: 회사 이메일 주소
- **직급**: 직원의 현재 직급 (예: Senior Developer, Principal Engineer)
- **부서**: 소속 부서 (예: Engineering, Product)
- **포지션**: 세부 직무 (예: Backend Developer, Frontend Developer)

**3단계: 기술 스택 입력**
- **기술 스택** 필드에 보유 기술을 입력합니다
- 쉼표(,)로 구분하여 여러 기술을 입력할 수 있습니다
- 예시: `Java, Spring Boot, AWS, Docker, Kubernetes`

**4단계: 경력 정보 입력**
- **경력 연수**: 총 경력 연수를 숫자로 입력합니다
- 예시: `5` (5년 경력)

**5단계: 등록 완료**
- 모든 정보를 입력한 후 **"등록"** 버튼을 클릭합니다
- 성공 메시지가 표시되고 직원 목록이 자동으로 새로고침됩니다
- 새로 등록된 직원이 목록에 나타납니다

#### 입력 예시

```
이름: 홍길동
이메일: hong.gildong@company.com
직급: Senior Developer
부서: Engineering
포지션: Backend Developer
기술 스택: Python, Django, PostgreSQL, Redis, Docker
경력 연수: 5
```

#### 주의사항

- 이메일 주소는 중복될 수 없습니다
- 기술 스택은 정확한 명칭으로 입력해야 추천 시스템이 올바르게 작동합니다
- 등록 후에는 시스템 관리자만 정보를 수정할 수 있습니다

## 프로젝트 관리

### 프로젝트 목록 조회

1. 좌측 메뉴에서 **"프로젝트 관리"**를 클릭합니다
2. 등록된 모든 프로젝트 목록이 표시됩니다
3. 각 프로젝트 카드에는 다음 정보가 표시됩니다:
   - 프로젝트 명
   - 산업 분야
   - 필요 기술 스택
   - 팀 규모
   - 프로젝트 상태

### 새로운 프로젝트 등록

#### 단계별 가이드

**1단계: 등록 모달 열기**
- 프로젝트 관리 페이지 우측 상단의 **"+ 신규 프로젝트 등록"** 버튼을 클릭합니다
- 프로젝트 등록 모달 창이 나타납니다

**2단계: 프로젝트 기본 정보 입력**

필수 입력 항목:
- **프로젝트명**: 프로젝트의 명칭
- **산업 분야**: 프로젝트가 속한 산업 (예: Finance / Banking, E-commerce, Healthcare)
- **필요 기술 스택**: 프로젝트에 필요한 기술들
- **팀 규모**: 필요한 인원 수
- **프로젝트 기간**: 예상 소요 기간 (개월)
- **시작 예정일**: 프로젝트 시작 날짜

**3단계: 상세 정보 입력 (선택사항)**
- **프로젝트 설명**: 프로젝트에 대한 상세 설명
- **예산 규모**: 프로젝트 예산
- **클라이언트 정보**: 클라이언트 회사명

**4단계: 등록 완료**
- 모든 정보를 입력한 후 **"등록"** 버튼을 클릭합니다
- 성공 메시지가 표시되고 프로젝트 목록이 자동으로 새로고침됩니다

#### 입력 예시

```
프로젝트명: 모바일 뱅킹 앱 개발
산업 분야: Finance / Banking
필요 기술 스택: React Native, Node.js, MongoDB, AWS
팀 규모: 8
프로젝트 기간: 12개월
시작 예정일: 2025-12-01
```

## 인력 추천

### AI 인력 추천 받기

프로젝트에 최적의 인력을 AI가 자동으로 추천해드립니다.

#### 단계별 가이드

**1단계: 추천 시작**
- 프로젝트 관리 페이지에서 원하는 프로젝트 카드를 찾습니다
- 프로젝트 카드 하단의 **"AI 인력 추천 받기"** 버튼을 클릭합니다

**2단계: 추천 결과 확인**
- 시스템이 자동으로 최적의 인력을 분석합니다 (약 5-10초 소요)
- 추천 결과 페이지로 이동합니다
- 추천된 인력 목록이 점수 순으로 표시됩니다

**3단계: 추천 결과 분석**

각 추천 후보자 카드에는 다음 정보가 표시됩니다:

- **종합 점수**: 전체 적합도 점수 (0-100점)
- **기술 매칭 점수**: 요구 기술과의 일치도
- **친밀도 점수**: 팀원들과의 협업 이력
- **가용성**: 현재 배정 가능 여부
- **보유 기술**: 후보자의 기술 스택
- **AI 추천 근거**: Claude AI가 생성한 추천 이유

**4단계: 상세 정보 확인**
- 후보자 카드의 **"상세 정보 보기"** 버튼을 클릭합니다
- 다음 정보를 확인할 수 있습니다:
  - 전체 프로젝트 이력
  - 기술별 숙련도 및 경력
  - 팀원들과의 친밀도 상세 분석
  - 과거 프로젝트 성과


### 프로젝트에 인력 배정하기

추천받은 인력을 실제 프로젝트에 배정하는 방법입니다.

#### 단계별 가이드

**1단계: 배정할 인력 선택**
- 추천 결과 페이지에서 배정하고 싶은 후보자를 선택합니다
- 후보자 카드의 **"프로젝트에 투입"** 버튼을 클릭합니다

**2단계: 배정 정보 입력**
- 프로젝트 배정 모달 창이 나타납니다
- 다음 정보를 입력합니다:
  - **역할**: 프로젝트에서의 역할 (예: Lead Developer, Backend Developer)
  - **시작일**: 투입 시작 날짜
  - **투입률**: 프로젝트 투입 비율 (0-100%, 기본값: 100%)

**3단계: 배정 확인**
- 입력한 정보를 확인합니다
- **"배정 확인"** 버튼을 클릭합니다

**4단계: 배정 완료**
- 성공 메시지가 표시됩니다
- 해당 직원의 상태가 "배정됨"으로 변경됩니다
- 프로젝트 팀 구성원 목록에 추가됩니다

#### 배정 시 주의사항

- 이미 다른 프로젝트에 100% 투입된 직원은 배정할 수 없습니다
- 투입률이 100% 미만인 경우, 여러 프로젝트에 동시 배정이 가능합니다
- 배정 후에는 프로젝트 관리자 승인이 필요할 수 있습니다

#### 추천 점수 이해하기

**종합 점수 (Overall Score)**
- 기술 매칭, 친밀도, 가용성을 종합한 최종 점수
- 90점 이상: 매우 적합
- 80-89점: 적합
- 70-79점: 보통
- 70점 미만: 추가 검토 필요

**기술 매칭 점수 (Skill Match Score)**
- 프로젝트 요구 기술과 후보자 보유 기술의 일치도
- 필수 기술 보유 여부와 숙련도를 고려

**친밀도 점수 (Affinity Score)**
- 기존 팀원들과의 협업 이력 및 커뮤니케이션 패턴 분석
- 과거 프로젝트 협업 경험, 메신저 소통 빈도 등을 종합

**가용성 점수 (Availability Score)**
- 현재 프로젝트 배정 상태
- 100점: 즉시 투입 가능
- 50점: 부분 투입 가능
- 0점: 투입 불가

## 평가 관리

### 이력서 업로드 및 자동 분석

새로운 직원이나 프리랜서의 이력서를 업로드하면 AI가 자동으로 분석합니다.

#### 단계별 가이드

**1단계: 평가 관리 페이지 접속**
- 좌측 메뉴에서 **"평가 관리"**를 클릭합니다
- 평가 대기 중인 직원 목록이 표시됩니다

**2단계: 이력서 업로드**
- 평가할 직원의 카드에서 **"이력서 업로드"** 버튼을 클릭합니다
- 이력서 업로드 모달이 나타납니다
- **"파일 선택"** 버튼을 클릭하여 PDF 파일을 선택합니다
- **"업로드"** 버튼을 클릭합니다

**3단계: 자동 분석 대기**
- 시스템이 자동으로 이력서를 분석합니다 (약 30초-1분 소요)
- 분석 과정:
  1. AWS Textract로 텍스트 추출
  2. Claude AI로 구조화된 정보 추출
  3. 정량적 분석 수행 (경력, 기술 스택, 프로젝트 수)
  4. 정성적 분석 수행 (강점, 약점, 적합 프로젝트)

**4단계: 분석 결과 확인**
- 분석이 완료되면 평가 결과가 표시됩니다
- 다음 정보를 확인할 수 있습니다:

**정량적 평가 결과**:
- 경력 점수
- 프로젝트 수
- 기술 다양성
- 기술 스택 최신성
- 종합 점수

**정성적 평가 결과**:
- 강점 (Strengths)
- 약점 (Weaknesses)
- 적합한 프로젝트 유형
- 개발이 필요한 영역
- AI 신뢰도 점수

### 평가 워크플로우

평가 결과에 대한 승인, 검토, 반려 프로세스입니다.

#### 평가 상태

- **대기중 (Pending)**: 평가가 완료되어 검토 대기 중
- **승인됨 (Approved)**: 평가가 승인되어 확정됨
- **검토중 (Under Review)**: 추가 검토가 필요한 상태
- **반려됨 (Rejected)**: 평가가 반려되어 재평가 필요

#### 평가 승인하기

**1단계: 평가 결과 검토**
- 평가 관리 페이지에서 "대기중" 상태의 평가를 확인합니다
- 평가 내용이 정확한지 검토합니다

**2단계: 승인 처리**
- 평가 카드의 **"승인"** 버튼을 클릭합니다
- 승인 확인 모달이 나타납니다
- 승인 코멘트를 입력합니다 (선택사항)
- **"승인 확인"** 버튼을 클릭합니다

**3단계: 승인 완료**
- 평가 상태가 "승인됨"으로 변경됩니다
- 해당 직원이 정식으로 시스템에 등록됩니다
- 프로젝트 추천 대상에 포함됩니다

#### 평가 검토 요청하기

추가 검증이 필요한 경우 검토를 요청할 수 있습니다.

**1단계: 검토 요청**
- 평가 카드의 **"검토"** 버튼을 클릭합니다
- 검토 요청 모달이 나타납니다

**2단계: 검토 사항 입력**
- **검토 코멘트**: 검토가 필요한 이유를 입력합니다
- **검토 포인트**: 구체적으로 확인이 필요한 항목을 입력합니다
  - 예: "프로젝트 성과 검증", "기술 스택 확인", "경력 기간 검증"

**3단계: 검토 요청 완료**
- **"검토 요청"** 버튼을 클릭합니다
- 평가 상태가 "검토중"으로 변경됩니다
- 담당자에게 알림이 전송됩니다

#### 평가 반려하기

평가 내용이 부적절한 경우 반려할 수 있습니다.

**1단계: 반려 처리**
- 평가 카드의 **"반려"** 버튼을 클릭합니다
- 반려 사유 입력 모달이 나타납니다

**2단계: 반려 사유 입력**
- **반려 사유**: 반려하는 구체적인 이유를 입력합니다
- **필요 조치**: 재평가를 위해 필요한 조치를 입력합니다
  - 예: "평가 기준 재정의", "추가 데이터 수집", "이력서 재업로드"

**3단계: 반려 완료**
- **"반려 확인"** 버튼을 클릭합니다
- 평가 상태가 "반려됨"으로 변경됩니다
- 담당자에게 반려 사유가 전달됩니다

## 도메인 분석

### 신규 도메인 확장 기회 분석

조직의 프로젝트 이력을 분석하여 진출 가능한 신규 사업 도메인을 발견합니다.

#### 단계별 가이드

**1단계: 도메인 분석 페이지 접속**
- 좌측 메뉴에서 **"도메인 분석"**을 클릭합니다
- 도메인 분석 대시보드가 표시됩니다

**2단계: 분석 실행**
- 페이지 상단의 **"도메인 분석 실행"** 버튼을 클릭합니다
- 분석 유형을 선택합니다:
  - **신규 도메인 발굴**: 진출 가능한 새로운 도메인 식별
  - **확장 전략 수립**: 기존 도메인의 확장 전략 분석

**3단계: 분석 결과 확인**

분석이 완료되면 다음 정보가 표시됩니다:

**현재 도메인 포트폴리오**:
- 조직이 현재 보유한 도메인 목록
- 각 도메인별 프로젝트 수
- 투입 인력 수
- 핵심 기술 스택

**신규 도메인 기회**:
- AI가 식별한 진출 가능한 도메인
- 각 도메인별 정보:
  - 도메인명
  - 실현 가능성 점수 (0-100점)
  - 필요 기술 스택
  - 현재 보유 기술
  - 기술 갭 (부족한 기술)
  - 추천 팀 구성
  - 진입 전략
  - 예상 소요 기간
  - 리스크 수준

**4단계: 상세 분석 확인**
- 관심 있는 도메인 카드를 클릭합니다
- 상세 정보 모달이 나타납니다:
  - 시장 기회 분석
  - 경쟁사 현황
  - 필요 역량 상세
  - 단계별 진입 계획
  - 예상 투자 비용
  - ROI 예측

#### 도메인 분석 활용 방법

**전략적 의사결정**:
- 실현 가능성 점수가 80점 이상인 도메인을 우선 검토
- 기술 갭이 적고 기존 인력으로 커버 가능한 도메인 선택
- 시장 수요와 조직의 강점을 고려한 선택

**팀 구성 계획**:
- 추천 팀 구성을 참고하여 인력 배치 계획 수립
- 필요한 교육 프로그램 기획
- 외부 전문가 영입 계획 수립

**리스크 관리**:
- 리스크 수준이 높은 도메인은 파일럿 프로젝트로 시작
- 단계적 진입 전략 수립
- 정기적인 진행 상황 모니터링

## 대시보드

### 실시간 현황 모니터링

조직의 인력 및 프로젝트 현황을 한눈에 파악할 수 있습니다.

#### 대시보드 구성 요소

**1. 주요 메트릭 카드**

- **총 직원 수**: 시스템에 등록된 전체 직원 수
- **활성 프로젝트**: 현재 진행 중인 프로젝트 수
- **대기 중인 평가**: 검토가 필요한 평가 건수
- **가용 인력**: 현재 배정 가능한 직원 수

각 메트릭 카드에는 전월 대비 증감률이 표시됩니다.

**2. 기술 스택 분포**

- 조직 내 기술 스택별 보유 인력 수를 차트로 표시
- 가장 많이 보유한 기술 Top 10
- 기술별 평균 숙련도
- 신규 트렌드 기술 보유 현황

**3. 도메인 커버리지**

- 프로젝트 수행 도메인별 분포
- 도메인별 투입 인력 수
- 도메인별 매출 기여도
- 성장 중인 도메인 하이라이트

**4. 프로젝트 현황**

- 진행 중인 프로젝트 목록
- 프로젝트별 진행률
- 마감 임박 프로젝트 알림
- 인력 부족 프로젝트 경고

**5. 인력 가용성**

- 가용 인력 목록
- 투입 예정 인력
- 프로젝트 종료 예정 인력
- 휴가/교육 중인 인력

#### 대시보드 활용 팁

**일일 모니터링**:
- 매일 아침 대시보드를 확인하여 전체 현황 파악
- 대기 중인 평가가 있는지 확인
- 긴급 알림이나 경고 사항 확인

**주간 리뷰**:
- 주간 단위로 메트릭 변화 추이 분석
- 기술 스택 분포 변화 모니터링
- 프로젝트 진행 상황 점검

**월간 보고**:
- 월말에 대시보드 데이터를 기반으로 보고서 작성
- 도메인별 성과 분석
- 인력 활용률 분석
- 다음 달 계획 수립

## 자주 묻는 질문 (FAQ)

### 직원 관리 관련

**Q: 직원 정보를 수정하려면 어떻게 해야 하나요?**
A: 현재 버전에서는 시스템 관리자만 직원 정보를 수정할 수 있습니다. 수정이 필요한 경우 관리자에게 문의하세요.

**Q: 기술 스택을 입력할 때 정확한 명칭을 모르겠어요.**
A: 일반적으로 사용되는 기술 명칭을 입력하면 시스템이 자동으로 정규화합니다. 예: "자바" → "Java", "스프링부트" → "Spring Boot"

**Q: 퇴사한 직원은 어떻게 처리하나요?**
A: 시스템 관리자가 직원 상태를 "비활성"으로 변경할 수 있습니다. 비활성 직원은 추천 대상에서 제외됩니다.

### 프로젝트 관리 관련

**Q: 프로젝트를 삭제할 수 있나요?**
A: 진행 중인 프로젝트는 삭제할 수 없습니다. 프로젝트를 종료 상태로 변경한 후 관리자가 삭제할 수 있습니다.

**Q: 한 직원을 여러 프로젝트에 배정할 수 있나요?**
A: 네, 가능합니다. 투입률을 조정하여 여러 프로젝트에 동시 배정할 수 있습니다. 단, 총 투입률이 100%를 초과할 수 없습니다.

**Q: 프로젝트 정보를 수정하려면?**
A: 프로젝트 카드의 "수정" 버튼을 클릭하여 정보를 업데이트할 수 있습니다.

### 인력 추천 관련

**Q: 추천 결과가 마음에 들지 않아요.**
A: 추천 우선순위를 변경해보세요. "기술 중심", "친밀도 중심", "균형" 중 선택할 수 있습니다. 또한 필요 기술 스택을 더 구체적으로 입력하면 정확도가 향상됩니다.

**Q: 추천 점수는 어떻게 계산되나요?**
A: 기술 매칭 점수, 친밀도 점수, 가용성 점수를 가중 평균하여 계산됩니다. 각 점수의 가중치는 선택한 우선순위에 따라 달라집니다.

**Q: 추천 결과를 저장할 수 있나요?**
A: 네, 추천 결과는 자동으로 저장되며 나중에 다시 확인할 수 있습니다.

### 평가 관리 관련

**Q: 지원되는 이력서 파일 형식은?**
A: 현재 PDF 형식만 지원합니다. Word 파일은 PDF로 변환 후 업로드해주세요.

**Q: 이력서 분석이 실패했어요.**
A: 이력서 파일이 손상되었거나 텍스트 추출이 불가능한 형식일 수 있습니다. 다른 PDF 뷰어로 파일을 열어 확인 후 재업로드해주세요.

**Q: AI 분석 결과가 부정확해요.**
A: 반려 기능을 사용하여 재평가를 요청하세요. 반려 사유에 부정확한 부분을 구체적으로 명시하면 도움이 됩니다.

### 도메인 분석 관련

**Q: 도메인 분석은 얼마나 자주 실행해야 하나요?**
A: 분기별로 실행하는 것을 권장합니다. 신규 프로젝트가 많이 추가되거나 조직 구조가 변경된 경우 즉시 실행하세요.

**Q: 실현 가능성 점수는 어떻게 계산되나요?**
A: 현재 보유 기술과 필요 기술의 갭, 시장 수요, 조직의 역량, 예상 투자 비용 등을 종합적으로 고려하여 AI가 계산합니다.

## 문제 해결

### 로그인 문제

**증상**: 로그인이 되지 않습니다.
**해결 방법**:
1. AWS IAM 자격 증명이 올바른지 확인
2. 브라우저 캐시 및 쿠키 삭제
3. 다른 브라우저로 시도
4. 시스템 관리자에게 계정 상태 확인 요청

### 페이지 로딩 문제

**증상**: 페이지가 로딩되지 않거나 매우 느립니다.
**해결 방법**:
1. 인터넷 연결 상태 확인
2. 브라우저 새로고침 (F5 또는 Ctrl+R)
3. 브라우저 캐시 삭제
4. 다른 브라우저로 시도

### 파일 업로드 문제

**증상**: 이력서 파일 업로드가 실패합니다.
**해결 방법**:
1. 파일 크기가 10MB 이하인지 확인
2. PDF 형식인지 확인
3. 파일명에 특수문자가 없는지 확인
4. 다른 PDF 뷰어로 파일을 열어 손상 여부 확인

### API 오류

**증상**: "500 Internal Server Error" 메시지가 표시됩니다.
**해결 방법**:
1. 잠시 후 다시 시도
2. 입력한 데이터가 올바른지 확인
3. 문제가 지속되면 시스템 관리자에게 문의
